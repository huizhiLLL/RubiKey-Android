# RubiKey 路线图

日期：`2026-07-15`

## 已收口决策

- 使用 Kotlin + Jetpack Compose 构建应用层和 UI。
- 成熟的 Java 协议代码保持 Java，通过明确接口与 Kotlin 互操作。
- 多品牌协议通过统一入口接入，业务层只识别标准转动事件。
- 全局手势采用用户显式启用的 `AccessibilityService`。
- 一个转动只对应一个点击、滑动或无动作。
- 首轮验收 Android 15、默认显示器和竖屏跑酷类游戏。
- 项目采用 GPLv3，不以应用商店上架为目标。

## 阶段一：工程基线

状态：已完成。

- 初始化 Git 和 Android 工程。
- 建立构建、文档、开发约束和许可证基线。
- 提供最小可启动 Compose 入口。

## 阶段二：Moyu32 POC

状态：已完成。

已完成统一协议入口、Moyu32 连接、动作映射、辅助功能手势和 Android 15 真机验证；端到端手势已被目标游戏接受。

## 阶段三：可分发稳定版

状态：已完成。

- 使用 RubiKey 独立签名密钥构建 `0.2.0`，`versionCode = 3`。
- 启用 R8 压缩，并提供侧载、覆盖升级和 GPLv3 源码交付说明。
- 使用自适应魔方图标和固定浅色黑白主题建立品牌基线。
- 完成首次侧载、同签名覆盖升级、受限设置和目标游戏输入验收。
- Release lint、构建、签名校验和 SHA-256 校验纳入发布门禁。

## 阶段四：多品牌扩展

状态：已完成。

- 已接入 GAN v2/v3/v4，保留独立 Service、AES 地址派生、批量通知和 history 补回逻辑。
- 已接入 QiYi QYSC/Tornado V4，保留广播协议地址、MTU、hello/ACK、CRC 和 history 时间序列逻辑。
- 三品牌共用标准转动、映射和辅助功能业务；扫描品牌参与 GATT 二次确认，避免重叠特征误判。
- 已完成三品牌在 Android 15 真机上的连接、电量、快速转动、补帧、后台运行和外部游戏输入验收。

## 阶段五：发布后体验完善

状态：按真实使用反馈决定。

- 动作映射导入导出和一键恢复默认。
- 扫描/连接超时与一键重新连接。
- Android 12-14 及不同厂商后台策略兼容矩阵。
- 根据需求决定是否增加配置分组、前台应用切换和横屏适配。
- 复杂宏、云同步和图像识别没有默认建设计划。
