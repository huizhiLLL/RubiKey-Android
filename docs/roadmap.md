# RubiKey 路线图

日期：`2026-07-14`

## 已收口决策

- 使用 Kotlin + Jetpack Compose 构建应用层和 UI。
- 成熟的 Java 协议代码保持 Java，通过明确接口与 Kotlin 互操作。
- POC 只实现 Moyu32，但多品牌协议入口从第一版成立。
- 全局手势采用用户显式启用的 `AccessibilityService`。
- 一个转动只对应一个点击、滑动或无动作。
- 首轮只验收 Android 15、默认显示器和竖屏跑酷类游戏。
- 项目采用 GPLv3，不以应用商店上架为目标。

## 阶段一：工程基线

状态：已完成。

- 初始化 Git 和 Android 工程。
- 建立构建、文档、开发约束和许可证基线。
- 提供最小可启动 Compose 入口。

## 阶段二：Moyu32 POC

状态：已完成。

已完成统一协议入口、Moyu32 连接、动作映射、辅助功能手势和 Android 15 真机验证；端到端手势已被目标游戏接受。

## 阶段三：可分发 Beta

状态：构建链路完成，待 Release APK 真机回归。

- 使用 RubiKey 独立签名密钥构建 `0.2.0-beta.1`。
- 启用 R8 压缩，并提供侧载、升级和 GPLv3 源码交付说明。
- 使用自适应魔方图标和固定浅色黑白主题建立首个品牌基线。
- 真机确认首次侧载、同签名覆盖升级、受限设置和目标游戏输入保持可用。

## 阶段四：多品牌扩展

状态：协议代码与 JVM 测试已完成，待多型号真机验收。

- 已接入 GAN v2/v3/v4，保留独立 Service、AES 地址派生、批量通知和 history 补回逻辑。
- 已接入 QiYi QYSC/Tornado V4，保留广播协议地址、MTU、hello/ACK、CRC 和 history 时间序列逻辑。
- 三品牌共用标准转动、映射和辅助功能业务；扫描品牌参与 GATT 二次确认，避免重叠特征误判。
- GAN v2/v3/v4 与 QiYi QYSC/Tornado V4 的连接、电量、快速转动、补帧、后台运行和外部游戏输入待 Android 15 真机验证。

## 阶段五：体验完善

状态：不属于当前 POC。

根据真实使用结果决定是否增加配置分组、前台应用切换、导入导出和横屏适配。复杂宏、云同步和图像识别没有默认建设计划。
